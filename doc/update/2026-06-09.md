# 2026-06-09 更新日志

## v1.3.0 - 自动化部署

### 部署脚本

- 新增 `deploy.sh` 一键部署脚本
- 支持两种部署模式：Nginx + pm2、Docker
- 自动检测并安装缺失依赖（pm2、Nginx）
- 首次安装与版本更新自动识别
- 更新时自动备份并恢复用户数据（配置、文章、图片）
- 支持指定版本部署：`--version v1.2.0`
- 支持自定义安装路径：`--path /opt/blog`

### Nginx 自动配置

- 脚本询问用户是否自动配置 Nginx
- 自动安装 Nginx（如果未安装）
- 询问域名并生成对应配置
- 支持 HTTP/HTTPS 配置选择
- 自动启用站点并重启 Nginx

### pm2 配置

- 新增 `ecosystem.config.js` pm2 配置模板
- 配置自动重启、日志输出、内存限制
- 支持开机自启

### GitHub Actions

- 新增 `.github/workflows/release.yml` 自动构建工作流
- 推送 tag（v*）自动触发构建
- 自动打包 `dist.tar.gz` 和 `backend.tar.gz`
- 自动创建 GitHub Release

### 文档

- 新增 `doc/deploy.md` 部署指南
  - 环境要求与依赖安装（Ubuntu/CentOS/Arch/macOS）
  - Nginx + pm2 部署流程
  - Docker 部署流程
  - 数据保护策略说明
  - 故障排查与备份恢复
- 新增 `doc/update/index.md` 更新日志索引
- 更新 `README.md`
  - 添加生产部署快速开始
  - 添加更新日志索引
  - 说明 `.example` 文件使用方式

### 其他

- 更新 `.gitignore` 添加 `logs/` 目录

## v1.3.1 - 配置修复与逻辑优化

### Bug 修复

- **docker-compose.yml**：修复 volumes 路径错误
  - 错误：`./data:/app/backend/data`
  - 正确：`./backend/data:/app/backend/data`
- **deploy.sh**：修复 `cp --parents` 在 macOS 上不兼容的问题
  - 改用 `while read` 循环实现跨平台兼容
- **deploy.sh**：移除冗余的 `.example` 文件复制逻辑
  - Release 打包时已处理 example → 实际文件，脚本中无需重复处理

### 逻辑优化

- **deploy.sh**：优化 Nginx 配置时机
  - 首次部署：询问用户是否配置 Nginx ✅
  - 版本更新：跳过 Nginx 配置（已存在）✅
  - 降级操作：跳过 Nginx 配置 ✅

### 文档更新

- **doc/deploy.md**：添加详细的配置文件说明
  - 配置文件位置
  - 首次部署必须修改的配置（JWT、密码）
  - 站点信息配置说明
  - 全部配置项参考表格

### CI/CD 修复

- **release.yml**：修复 GitHub Actions 兼容性问题
  - `softprops/action-gh-release@v1` → `@v2`
  - 添加 `permissions: contents: write` 权限配置
  - 添加 SHA256 校验文件生成

### 安全增强

- **deploy.sh**：添加 SHA256 文件完整性校验
  - 下载 `checksums-sha256.txt` 校验文件
  - 自动校验 `dist.tar.gz` 和 `backend.tar.gz`
  - 校验失败时终止部署，防止文件篡改

### 版本更新

- **package.json**：版本号更新为 `1.3.0`
