# 2026-06-10 更新日志

## v1.3.1 - 部署脚本重构与 Gitee 源支持

### 部署脚本重构

- **模式命名修正**：`--mode nginx` → `--mode pm2`，明确区分部署模式（pm2/docker）与反向代理（Nginx）
- **两种模式均支持 Nginx**：pm2 和 Docker 模式都可以配置 Nginx 反向代理
- **删除重复代码**：移除 main() 中三段重复的模式选择逻辑，合并为一段
- **下载重试机制**：新增 `download_with_retry` 函数，自动重试 3 次，间隔 5 秒
- **修复 `set -e` 导致静默退出**：下载失败时正确进入错误处理，不再直接退出脚本
- **修复差异比较失效**：`compare_files` 现在先解压压缩包再比较，之前因目录不存在导致始终输出"无文件变更"
- **修复更新后 pm2 异常**：`start_services` 检测到进程状态为 errored/stopped 时自动删除重建
- **修复 ecosystem.config.js 丢失**：更新后自动检查并重建 pm2 配置文件
- **自动安装 tsx**：pm2 模式下自动检测并安装 tsx 全局依赖
- **macOS Nginx 兼容**：配置路径、站点启用、重启命令适配 macOS（Homebrew）

### Gitee 源支持

- **新增 `--source` 参数**：支持 `gitee`（默认）和 `github` 两种下载源
- **国内服务器默认 Gitee**：下载速度更快，无需翻墙
- **Gitee API 兼容**：`get_latest_version` 和 `download_release` 支持两个平台
- **操作总览显示下载源**：部署时清晰显示当前使用的下载源

### CI/CD

- **新增 `sync-to-gitee.yml`**：GitHub Actions 自动同步 Release 到 Gitee
  - push 到 main 或打 tag → 自动同步代码
  - 发布 Release → 自动同步 Release 及附件
  - 支持 `workflow_dispatch` 手动触发指定版本同步

### 文档更新

- **doc/deploy.md**：
  - 所有 `Nginx + pm2 模式` → `pm2 模式`
  - 快速开始区分国内（Gitee）/海外（GitHub）下载路径
  - 版本更新命令新增 `--source` 参数说明
  - 手动部署部分同时列出两个下载源
- **README.md**：快速开始添加 Gitee 下载路径
- **doc/release-guide.md**：发布流程说明（已有）
